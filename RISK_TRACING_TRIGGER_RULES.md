# 重大风险溯源自动触发规则

这份规则解决的是：模型在什么情况下不能只做普通风险提示，而必须默认切换到“重大风险溯源模式”。

## 一、默认触发条件

只要满足以下任一条件，就默认触发“重大风险溯源模式”：

### 规则 A：命中单个强风险信号
- 退市
- 私有化
- ST / *ST / 退市风险警示
- 债务实质违约
- 债务展期
- 商票逾期
- 评级下调
- 立案调查
- 行政处罚
- 交易所监管工作函
- 持续经营重大不确定性
- 审计保留意见 / 无法表示意见

### 规则 B：命中两个以上中强度风险信号
以下信号任意同时命中 2 项及以上，也要自动触发：
- 控股股东高比例质押
- 控股股东冻结 / 被动减持
- 控制权变化
- 重大诉讼 / 被执行 / 失信
- 停产 / 减产 / 关厂
- 重大减值
- 流动性紧张 / 借新还旧
- 主流财经媒体集中负面舆情

### 规则 C：命中链式风险组合
如果出现以下组合词或事实链，必须直接切换到风险溯源模式：
- 退市 + 私有化
- 私有化 + 纾困
- 纾困 + 债务展期 / 债务违约
- 评级下调 + 债务展期 / 商票逾期
- 质押冻结 + 控制权变化
- 问询 / 处罚 + 财务异常 / 审计异常

## 二、触发后必须做什么

一旦触发，风险段不能只写一句“存在风险”，而必须至少完成以下动作：
- 在正文里单独给它一个完整小节
- 按“异常结果 -> 直接原因 -> 中层原因 -> 更深层机制 -> 回到今天”的顺序推进
- 区分 A/B/C 级证据来源
- 明确这件事是历史遗留，还是今天仍有效的风险变量

## 三、特别提醒：退市 / 私有化 / 纾困场景

如果文章识别到“退市、私有化、纾困、债务展期、违约”中的任意两项以上，不允许只写：
- 公司曾经退市
- 公司后来被私有化
- 公司存在债务问题

而是必须继续追问：
1. 退市或私有化是结果，直接触发是什么？
2. 为什么会走到纾困或展期这一步？
3. 哪一层债务、扩张、担保或治理机制先出了问题？
4. 有没有更深层的外部放大机制？
5. 今天这条链条还在不在？

## 四、禁止误触发的情形

以下情形不要轻易升级成重大风险溯源模式：
- 只有普通利润波动，没有信用或监管异常
- 只有单条媒体质疑，没有形成公开风险事件
- 只有市场杂音，没有公开资料或交叉验证
- 只有员工/招聘层面的零散声音

## 五、目标效果

自动触发规则的目标不是让文章变得更吓人，而是让真正重要的风险不会被轻描淡写带过。
真正触发时，文章要做到：
- 不停留在结果
- 能追到原因
- 能分清证据强弱
- 能回到今天的判断
