# 公众号「光影资本 Pro」爆款文章写作 Skill（v2.1）

基于「光影资本 Pro」公众号文章的深度拆解 + 2025-2026 年公众号爆款规律调研，形成的固定模板与自动化工作流。

**核心目标**：让 skill 每天自动生成有流量、能爆的文章，而不是机械的财报搬运工。

**默认执行语义（重要）**：
- 只要用户说“跑爆款文章”“跑一次选题，然后跑爆款文章”“按默认流程出稿”“深入分析”“帮我写一篇”，都默认走同一条固定流水线
- 如果用户只说“深入分析贵州百灵”“深入分析东方集团”这类“深入分析 + 公司名”的极简指令，也默认视为：深度分析该上市公司，走完整流水线，并在完成后自动推公众号草稿箱、打包 ZIP、回当前会话。除非用户明确说“不要推草稿箱”或“不要 ZIP”。
- 固定流水线：选题 → 财务分析 → 初稿 → `review.md` 一次性列出全部问题和逐条修改意见 → 打回重写 → 复审到“可发布” → 配图 → HTML → 草稿箱 → ZIP 交付包 → 发当前微信会话。若审稿连续 4 轮仍未通过，则停止继续改稿，直接基于最后一轮送审稿生成配图和 HTML，并把稿件、审稿意见、配图写入飞书文档，转人工审稿。
- 用户不需要每次重复强调审稿闭环；这是本 skill 的默认行为，不是额外要求，不得临时改成“自己写 API 代码再拼流程”。
- 当用户只给公司名而未写股票代码时，优先继续完整执行；能自动解析代码就直接跑，解析不到再补问，不能因为用户没写代码就退化成只做摘要或只聊天。
- **城投平台、地方国企、非上市控股公司（如「济南章丘控股集团」「XX城投」「XX国资运营」）一律走同一条流水线**：调用 write_article.js 不传股票代码 → 脚本自动检测无 A 股代码 → 走兼容模式 → 生成公司底稿 + 外部风险底稿 + 文章 + 配图 + 推草稿箱 + ZIP 打包。严禁 agent 自行判断企业性质后降维成手写报告。
- 判断规则：看到「深入分析 + 公司名」→ 直接调 write_article.js，不要先搜企业信息再手动写报告。
- **长任务必须先回执再执行**：调用 `write_article.js` 之前，先回当前会话一句短消息，例如“已开始分析，通常需要 5-15 分钟，我跑完就把草稿箱和 ZIP 结果回你”。不能让用户在长时间运行期间完全无反馈。
- **长任务必须读进度文件**：如果 `write_article.js` 运行超过 90 秒，必须读取输出目录里的 `generation_status.json`，并至少按阶段回一次进度（例如“正在审稿返修”“正在配图”“正在推草稿箱”）。不要一直静默等待到任务结束。
- **人工接管续跑时，最久每 3 分钟必须回一次进度**：如果主进程超时、LLM 调用超时、process poll 超时，或者进入手动续跑模式，必须在 3 分钟内给用户一条明确状态，例如“返修超时，我正在继续续跑剩余流程”“正在推草稿箱”“正在打包 ZIP”。绝不允许续跑阶段长时间静默。
- **人工接管续跑不能半途而废**：只要核心产物已经部分生成（如 `article.md`、`review.md`、`images/`、`article.html` 任一存在），就必须继续把剩余流程补完，直到草稿箱、ZIP、结果回会话全部完成，除非遇到明确不可恢复的外部错误（如公众号凭据错误、白名单错误、磁盘满、权限错误）。
- **续跑优先级固定**：先确认当前产物状态 → 补 `article.md` / `review.md` → 补图片 → 补 HTML → 推草稿箱 → 打包 ZIP → 发当前会话。不要因为中途某一步超时，就停在“我再试试”而不继续推进。
- **港股上市公司（如「泡泡玛特 09992.HK」）走完整流水线**：传入港股代码（如 `09992.HK`）→ `write_article.js` 识别港股后缀 → `financial_analysis.py` 自动切换 AKShare 港股数据源获取三大表 → 正常生成财务底稿 + 文章 + 审稿 + 配图 + 推草稿箱 + ZIP。
- 如果任务来自主 agent / operator / 运营推广场景，默认必须直接调用本 skill 已有脚本：financial_analysis.py、write_article.js、render_html.py、publish_to_wechat.js。
- 禁止为了完成任务临时编写飞书 API 客户端、公众号客户端或另起一套写稿/审稿框架；除非用户明确要求开发新功能。
- **严禁手写 `zip` / `cp` 命令自行打包 ZIP**，必须调用 `node scripts/package_output.js --output-dir <dir> --zip-path <path>`；手写打包会漏掉 financial_analysis.md 等核心分析文件。
- **完成交付后，必须显式回当前会话**：不能只依赖 session 自然镜像。至少要主动发一条简短文字结果（例如“已推草稿箱，ZIP 已发你”），然后再显式发送 ZIP 文件到当前微信/飞书会话；否则一旦请求被路由到新 session，用户侧可能看不到任何最终回应。

所有 API Key 统一通过环境变量管理，**不要硬编码到代码中**。

```bash
# 复制配置模板
cp .env.example .env
# 编辑 .env，填入你的真实 Token
vi .env
```

需要配置的 Token：

| 环境变量 | 用途 | 获取方式 |

## 二、爆款标题公式库（🔥 核心更新 v2.1 - 搜索+点击双优化）

### 核心原则

1. **去掉"第 N 期"自我视角** → 读者不关心这是第几期
2. **控制在 25 字以内** → 手机屏幕完整显示
3. **用爆款公式激发点击** → 打开率从 1.13% 提升至 5-8%
4. **🆕 前半段命中搜索词，后半段制造点击冲动** → 搜索流量 + 打开率双收

### 🔍 搜索+点击双优化框架（v2.1 新增，基于微信搜一搜官方标准）

**背景**：涨粉阶段，搜索流量 > 私域打开率。标题必须同时满足两个目标：
- **搜索命中**：用户搜"三花智控""商票贴现""寒武纪年报"时能找到你
- **点击转化**：搜到之后还想点开

**双优化公式**：`[搜索关键词 + 核心事实] + [爆款钩子]`

| 层级 | 作用 | 位置 | 示例 |
|------|------|------|------|
| 第一层：搜索锚点 | 让搜一搜找到你 | 标题前半段（前 12 字） | `寒武纪首次盈利20亿` |
| 第二层：点击钩子 | 让用户想点开 | 标题后半段 | `财报里藏着3个危险信号` |

**对比**：
- ❌ 纯爆款：`十年烧光50亿，这家芯片公司凭什么突然翻身？`（搜不到"寒武纪"）
- ❌ 纯搜索：`寒武纪2025年报深度分析`（没人想点）
- ✅ 双优化：`寒武纪首次盈利20亿，财报里藏着3个危险信号`（搜得到 + 想点）

**搜索关键词嵌入规则**：
1. **公司名必须出现在标题中**（全称或常用简称，不能用"这家公司""某龙头"替代）
2. **行业/概念关键词至少出现 1 个**（如"商票贴现""光伏""芯片""机器人"）
3. **关键词自然嵌入，禁止堆砌**（❌ "寒武纪芯片AI人工智能年报" → 会被搜一搜降权）
4. **标题必须表意完整**，不能为了悬念故意隐瞒主语（搜一搜明确打压此类标题）
5. **禁止欺骗性标题**（标题与内容不符 → 搜一搜严格作弊处理）

**搜一搜官方红线（摘自官方优化教程）**：
- ❌ 表意不明的标题党、震惊体 → 降权
- ❌ 标题堆砌关键词 → 降权
- ❌ 标题与文章内容不相关 → 降权
- ❌ 标题过短缺乏信息 / 标题过长被截断 → 降权
- ❌ 欺骗性标题（假冒入口等）→ 严格作弊处理

## 三、文章结构（内容结构规范 v3）

### 核心原则

以后文章统一执行一个目标：**版式结构更像成熟公众号成稿，内容密度和研究深度不低于当前最佳样稿。**

换句话说：
- **版式学公众号节奏**：好读、好扫读、好转发
- **内容保留研究感**：有逻辑、有判断、有数据支撑
- **不要写成百科词条**，也不要写成“只有气氛没有研究”的空心爆文

### 结构总纲

以后默认采用 **1 个开头 + 7 个主段 + 1 个结尾** 的结构：

1. 开头钩子：先抛事实反差，再点出本文要回答的问题
2. 第一主段：把公司定位和核心矛盾讲透
3. 第二主段：企业发展时间轴 + 行业地位（关键节点、排名变迁、规模锚点）
4. 第三主段：股东价值分级分析（优质股东深挖加持，问题股东深挖传导，普通股东一笔带过）
5. 第四主段：把业务逻辑翻译成人话，讲清怎么赚钱
6. 第五主段：穿透财务，讲清哪些数据真支持、哪些只是表面热闹
7. 第六主段：写竞争格局和行业位置，必须有对比感
8. 第七主段：写风险与未来，回答后续该盯什么
9. 结尾收束：用一句话总结判断，再抛互动问题

### 默认版式要求

**开头：**
- 前 200 字必须覆盖公司名、行业词、核心数据或核心现象
- 禁止一上来写成立时间、注册地、实控人
- 先写“发生了什么”，再写“为什么值得看”，最后写“这篇要拆什么”

**小标题：**
- 全文控制在 5 个主标题左右
- 不要写“一、二、三、四”编号标题
- 小标题必须像判断句或提问句，且尽量自然嵌入公司名、行业词、核心概念词

**段落：**
- 单段尽量控制在 2-4 行
- 连续 6 行以上的大段必须拆开
- 每个主段结尾都要有 1 句收束，让读者知道“这一段真正说明了什么”

**图片：**
- 默认 4 张图，分别服务于：开头、业务、财务、结尾
- 图片是阅读中继点，不是装饰
- 图片前后必须有文字承接，避免孤立悬空

### 内容深度要求

**第一主段：公司定位和矛盾**
- 不写成“公司简介”资料卡
- 要回答：它为什么会被市场关注？这次最值得拆的矛盾是什么？

**第二主段：业务翻译**
- 不要堆产品列表，要讲赚钱主线
- 必须回答：它真正卖的是什么？利润是怎么来的？护城河是长期能力还是阶段性红利？

**第五主段：财务穿透**
- 至少覆盖最新营收、净利润、毛利率、ROE、资产负债率、现金流中的 4 项
- 每个关键数字后面都尽量补一句“这意味着什么”
- 财务段至少产出 1 句有证据支撑的可截图判断句

**第六主段：竞争格局**
- 不要只写“行业空间广阔、竞争激烈”
- 要用对比告诉读者：它在行业里到底算老几，优势是否真有稀缺性

**第七主段：风险与未来**
- 风险要具体到场景，不能只写抽象名词
- 必须写清楚后续真正值得跟踪的公开信号
- 保持观察型口径，不写成“结论已定”

### 风险信息补强范围（新增）

为了让文章内容比普通财报解读更深入，允许引入以下互联网公开风险信号：
- 控股股东质押、冻结、减持、控制权变化
- ST、*ST、退市风险警示、立案调查、交易所问询、监管处罚
- 重大诉讼、被执行、失信、资产冻结、仲裁
- 商票逾期、承兑异常、评级下调、债务展期、违约信号
- 停产、减产、关厂、重大审计意见、重大减值、持续经营疑虑
- 主流财经媒体集中报道的重大负面舆情

这些信息只有在**公开可验证**、且与投资判断直接相关时，才允许写入正文。

### 重大风险溯源框架（新增）

只要文章中识别到以下重大风险信号，就不能只停留在“风险标签”层面，而必须做因果链下钻：
- 退市、私有化、ST、*ST、退市风险警示
- 债务违约、展期、评级下调、商票逾期
- 控股股东高比例质押、冻结、控制权变化
- 重大诉讼、被执行、失信、资产冻结
- 立案调查、交易所问询、监管处罚
- 重大审计意见、持续经营疑虑、停产减产、重大减值

### 最少下钻 4 步

写作时至少要回答：
1. 结果事实是什么
2. 直接原因是什么
3. 中层原因是什么
4. 更深层机制是什么

素材足够时，再补第 5 步：
5. 这个机制今天还在不在

### 证据分层写法

- A 级证据：公告、交易所文件、法院信息、评级报告、审计报告，可直接写事实
- B 级证据：主流财经媒体、多源公开报道，可写“公开报道显示”“多家媒体提到”
- C 级证据：市场讨论、地方舆情、社交平台线索，只能写“市场曾有一种解释”，不能写成公司已确认结论

### 正文表达要求（新增）

风险链的内部推理可以按“异常结果 / 直接原因 / 中层原因 / 更深层机制 / 回到今天”来组织，但**正文默认不要把这些词直接写成小标签**。

也就是说：
- 可以按这条链思考
- 但成稿要写成自然段推进，而不是报告式填空
- 除非用户明确要求报告体，否则不要出现“异常结果：”“直接原因：”“中层原因：”这种硬标签

优先写法：
- 先用一句话点出现象
- 再用2到4个自然段顺着因果往下走
- 让证据、判断、转折自然嵌进句子里

禁止写法：
- 五个粗体标签排排站
- 每段一个结论句，像答题卡
- 同一段里把结果、原因、结论硬塞满

### 目标要求

风险段要写成“因果链”，不是写成“风险清单”。
读者看完要知道：
- 这件事为什么发生
- 哪条链条把风险放大了
- 这条链条今天是否仍然有效

### 明确排除的风险素材

以下内容默认不纳入现在这套爆款公众号文章技能：
- 招聘信息、HC 冻结、岗位增减
- 员工讨薪、员工吐槽、工作体验、离职率传闻
- 脉脉、BOSS 直聘、猎聘等平台上的匿名或半匿名评价
- 缺乏公开证据支撑的短视频/社交媒体爆料

除非这些内容已经被公司公告、交易所文件或主流财经媒体正式确认，否则不要写进正文。

### 金句与解释力要求

- 每 600-800 字至少出现 1 句可截图判断句
- 金句必须服务于研究判断，不能只是情绪爽句
- 全文至少完成 3 次“专业概念翻译成人话”
- 全文至少完成 2 次“对比之后得出的结论”
- 至少出现 1 次“市场看法 vs 真实经营”的拆分

### 结尾要求

结尾必须完成三件事：
1. 用一句话给出最终判断
2. 写清后续真正决定表现的变量
3. 抛出有讨论价值的互动问题

### 明确禁止

- 禁止滑回 14 板块八股体
- 禁止把文章写成连续长文，没有节奏停顿
- 禁止只追求版式漂亮，内容却像资料卡
- 禁止只追求金句密集，结果判断失真或支撑不足
- 禁止把文章写成单票推荐或荐股文案

### 自检问题

发稿前至少问自己：
- 这篇文章看上去像成熟公众号成稿，而不是普通研究长文吗？
- 开头是否同时完成了“事实反差 + 核心问题”两个任务？
- 中段是否在解释逻辑，而不是罗列信息？
- 财务段是否做到了“数据 + 翻译 + 判断”？
- 结尾是否是观点收束，而不是礼貌结束？

### 市场观察型爆文模板（明日起默认采用）

以“首航新能”合规版为基准，今后的财经公众号文章默认采用 **市场观察型爆文** 口径，而不是“单票推荐”口径。

**默认写法：**
- 开头先写 **反常识现象 + 事实边界**：先说市场发生了什么，再说哪些已经公告、哪些只是预期
- 全文只围绕 **一个核心矛盾** 展开，不要贪多，例如“题材热度跑在基本面前面”
- 多用“市场在看什么 / 市场在交易什么 / 更值得跟踪的是什么”这类观察句式
- 可以写冲突、反差、风险，但必须写成 **公开信息观察**，不要写成“结论已定”
- 文章目标是让读者觉得“这篇看完更明白发生了什么”，不是让读者觉得“作者在带节奏”

**推荐结构：**
1. 现象钩子：股价、消息、行业异动里的反差
2. 事实边界：截至当日，哪些是公告确认，哪些只是市场预期
3. 业务翻译：公司到底做什么，为什么会被联想到这条主线
4. 财务穿透：财报哪里支持，哪里不支持
5. 观察结论：后续真正值得跟踪的公开信号是什么

**禁止滑回旧写法：**
- 不要把文章写成“这家公司值不值得买”
- 不要把文章写成“它就是核心受益标的”
- 不要用大量同行 PE 或市值比较来硬推结论，除非当日数据已核实且确有必要

### 合规性原则（⚠️ 极其重要！违反可能导致投诉删文）

上市公司拥有法务团队，会主动监控自媒体文章并投诉。以下规则**必须严格遵守**：

1. **事实信息必须来自数据源**
   - 实控人、股东、注册地、上市时间 → 必须从 `stock_basic` API 或 `financial_data.json` 获取
   - 如果数据源没有，写"据公开资料显示"并留空，由人工核实后补充
   - ❌ 严禁凭记忆写"创始人为 XXX"

2. **负面表述必须降级**
   - ❌ "大股东出逃" → ✅ "大股东近期有所减持"
   - ❌ "利润崩了" → ✅ "利润出现明显下降"
   - ❌ "赔本赚吆喝" → ✅ "增收不增利的情况值得关注"
   - ❌ "这家公司在硬撑" → ✅ "公司面临一定经营压力"

3. **平衡性原则**
   - 每个负面观点后面**必须跟一个正面或中性的解读**
   - 不能通篇负面，也不能通篇吹捧
   - 示例："毛利率有所下降（负面）→ 但公司正在通过海外建厂降低长期成本（正面）"

4. **禁止投资建议**
   - ❌ "建议买入/卖出"
   - ❌ "现在是抄底的好时机"
   - ✅ "以上分析仅供参考，投资者需根据自身情况独立判断"

5. **免责声明**（由 `generate_html.js` 自动追加到 HTML 底部，文章 Markdown 中**不要手动写**，避免出现两次）
   ```
   本文不构成任何投资建议，市场有风险，投资需谨慎。
   ```

### 财经号合规写法清单（发布前必须逐条过）

以下清单既是写作规则，也是审稿硬门槛。任何一条不满足，都不能进入"可发布"。

**A. 标题与摘要**
- 只能写"预期、观察、市场在看什么"，不能把未公告事项写成"订单、实锤合作、核心受益"
- 不得出现"暴涨在即、还有空间、值得布局、抄底时机、翻倍逻辑"
- 标题优先使用：`市场在看什么？`、`为什么会这样？`、`公开信息里真正能确认的是什么？`
- 🆕 标题前半段必须包含公司名和核心关键词（搜索命中），后半段用爆款钩子（点击转化）
- 🆕 禁止纯情绪词开头（如"诡异！""疯狂！"），搜一搜视为标题党会降权

**B. 事实边界**
- 只要公司没有公告，就必须写清"截至当日未见公司正式公告确认"
- 传闻、媒体消息、市场联想，必须明确标注为传闻/消息/预期，不能改写成事实
- 客户、订单、合作、份额、核心供应商等表述，若无正式依据，不能下结论

**C. 观点边界**
- 允许写：`市场在交易什么`、`更稳妥的理解是`、`后续值得跟踪的是`
- 禁止写：`建议买入/卖出`、`可以关注机会`、`后面还有上涨空间`、`就是龙头`
- 不对未来涨跌做确定性判断，不给目标价，不做收益承诺

**D. 风险呈现**
- 必须同时写支撑因素和不确定因素
- 每个强判断后面必须有数据或公开资料支撑
- 能写"需要继续观察"，就不要写"已经成立"

**E. 发布前审稿**
- `review.md` 必须单独包含"财经号合规写法清单"小节
- 该小节至少逐条检查：标题、摘要、导语、客户/订单/合作表述、风险表述、结尾互动句
- 只要出现一处把预期写成事实、一处明显荐股导向、一处无来源的强判断，就必须退回重写

## 四、爆款写作技巧（🔥 核心更新 v2.0）

### 1. 钩子开头（前 100 字定生死）

**目标**：用极强的数据反差或反直觉现象抓住眼球，引出主角（公司）。

**6 种钩子模板，根据公司特点选用：**

| 类型 | 模板 | 示例 |
|------|------|------|
| 反常识 | 所有人都觉得 XX，但实际上... | "所有人都觉得茅台卖的是酒，但实际上，茅台卖的是中国的面子经济。" |
| 数据冲击 | 一个数字：XX。什么概念？... | "一个数字：862 亿净利润。什么概念？相当于每天净赚 2.36 亿，每秒进账 2731 元。" |
| 悬念式 | 有一家公司，XX 却 XX... | "有一家公司，原材料成本不到售价的 9%，却让全中国的有钱人排队抢购。" |
| 对比式 | XX 比 YY 还 ZZ... | "茅台的毛利率比苹果还高，但它只是一家卖酒的。" |
| 场景代入 | 如果你 XX，你会发现... | "如果你去参加任何一场高端商务局，桌上一定有一样东西--飞天茅台。" |
| 灵魂拷问 | 一个问题：XX？ | "一个问题：增速掉到 6% 的茅台，还配得上 2 万亿市值吗？" |

**🔥 爆款开头公式（v2.0 新增）**：

```
【第 1 句】反直觉现象/数据反差（20-30 字）
【第 2 句】追问原因/制造悬念（15-25 字）
【第 3 句】预告文章价值（20-30 字）
```

**示例（寒武纪）**：
> **2025 年，一家连续亏损十年的公司突然赚了 20 亿。但诡异的是，年报发布后股价不涨反跌。**
>
> 市场到底在担心什么？
>
> **翻开寒武纪的财报，藏着 3 个危险信号。**

**预期效果**：完读率从 35% 提升至 **50%+**（进入大流量池的硬门槛）

### 2. 常见写作场景的爆款写法

**场景示例：公司介绍** - 不是写百科，是建立第一印象
- ❌ "XX 公司注册地位于 XX，成立于 XX 年"
- ✅ 先用钩子开头 → 再用 1-2 句话交代基本信息 → 最后一句"一句话定位"
- 金句示例："在白酒这个行业里，茅台不是第一名，而是唯一。"

**场景示例：股东结构** - 要有判断，不是罗列
- ❌ "前十大股东分别是 A 持有 X%，B 持有 Y%..."
- ✅ 先说结论（"股权高度集中且稳定"），再用 1-2 个细节佐证，最后给判断
- 关注点：有没有大比例质押？有没有频繁减持？这些比列数据更有价值

**场景示例：股价与估值** - 要有画面感
- ❌ "当前 PE 为 25 倍"
- ✅ "25 倍 PE 贵不贵？对比来看，五粮液 20 倍，泸州老窖 18 倍。市场愿意为茅台多付 25% 的溢价，买的是什么？是确定性。"
- 技巧：**把估值变成"买的是什么"的讨论**

**场景示例：核心业务** - 这是全文最重要的部分
- 用**"翻译"思维**：把商业模式翻译成普通人能懂的语言
- 必须回答：这家公司靠什么赚钱？为什么别人抢不走？
- 技巧：用**类比**（"茅台之于白酒，就像爱马仕之于包包"）

**场景示例：竞争格局** - 要有冲突感
- ❌ "行业竞争激烈，公司位于第一梯队"
- ✅ 用**VS 对决**的写法："茅台 vs 五粮液：一个是皇帝，一个是宰相"
- 技巧：给出**具体差距数字**，让读者感受量级

**场景示例：财务分析** - 数据要说人话
- 每个指标后面必须跟一句**"翻译"**
- ❌ "资产负债率 12.81%"
- ✅ "资产负债率 12.81%--说白了，茅台几乎不借钱做生意，账上现金多到花不完"
- 技巧：**先给评价标签（安全/危险/优秀/一般），再解释为什么**

**场景示例：行业前景** - 要有前瞻性判断
- 不是抄行业报告，是给出**你的观点**
- 必须包含：1 个利好趋势 + 1 个风险趋势 + 你的判断

**板块 13：投资风险** - 要真诚，不是走形式
- ❌ "存在政策风险、市场风险、经营风险"（废话三连）
- ✅ 每个风险要**具体到场景**："如果经济继续下行，商务宴请预算砍半，茅台的终端需求可能下降 20%"

**板块 14：结尾** - 留下值得转发的话
- 结尾 = 核心观点的浓缩 + 引发讨论的问题
- 模板："总的来说，XX 是一家 [判断] 的公司。[核心观点一句话]。你觉得呢？欢迎评论区聊聊。"

### 3. 金句密度要求（🔥 核心指标）

**每 500 字至少 1 句"值得截图"的话。**

**金句加粗规则：每句金句必须用 Markdown `**` 加粗标记，确保在公众号中以黑体高亮显示，方便读者截图传播。**
金句应覆盖：核心矛盾揭示、财务判断、风险定性、关键对比、结尾收束等关键位置。

金句类型：

- **类比金句**："茅台的护城河不是技术，不是资金，而是 70 年积累的文化认同--这东西，花多少钱都买不来。"
- **数据金句**："每卖出一瓶 1500 块的茅台，原材料成本不到 135 块。这不是做生意，这是印钞。"
- **反转金句**："茅台最大的风险不是竞争对手，而是年轻人不喝白酒了。"
- **总结金句**："买茅台的人，买的不是增长，而是确定性。"

### 4. 情绪节奏设计

全文情绪不能平铺直叙，要有**起伏**：

```
开头：惊艳/好奇（钩子）
  ↓
公司 + 业务：佩服/震撼（这家公司真牛）
  ↓
财务分析：理性/冷静（用数据说话）
  ↓
风险分析：警觉/担忧（但也有问题）
  ↓
结尾：思考/讨论（你怎么看？）
```

### 5. 互动引导模板（🔥 算法权重提升关键）

**文末必须设置具体互动问题**，提升评论率（评论率影响算法推荐权重）：

**3 种互动模板**：

| 类型 | 模板 | 示例 |
|------|------|------|
| 选择题 | 这 3 个风险，你最担心哪个？ | "寒武纪的 3 个风险--存货积压、股东减持、客户集中，你觉得哪个最致命？评论区聊聊。" |
| 观点题 | 你怎么看 XX 的未来？ | "十年亏损后首次盈利，你觉得寒武纪的翻身仗能持续吗？评论区告诉我。" |
| 经历题 | 你踩过 XX 的坑吗？ | "这 3 个财务陷阱，你踩过哪个？评论区聊聊你的经历。" |

**预期效果**：评论率提升 **60%**，直接影响算法推荐权重

### 6. 结尾模板（3 选 1）

**A. 判断型**（最推荐）
> "总的来说，XX 是一家 [优势] 突出但 [风险] 值得关注的公司。[一句话核心判断]。自然收束，可抛互动问题或开放性判断，不要固定结束语。"

**B. 提问型**（适合有话题性的公司）
> "[核心观点的提炼]。你怎么看 XX 的未来？评论区告诉我。自然收束，可抛互动问题或开放性判断，不要固定结束语。"

**C. 金句型**（适合龙头公司）
> "[一句定性金句]。这就是 XX，[一句话总结]。自然收束，可抛互动问题或开放性判断，不要固定结束语。"

**⚠️ 注意：免责声明由 `generate_html.js` 自动追加到 HTML 底部，文章 Markdown 中不要写，避免出现两次。**

## 五、完整工作流（唯一入口：write_article.js）

**⚠️ 强制规则：所有文章生成任务，必须且只能通过调用 `write_article.js` 完成。禁止分步手动执行。**

### write_article.js 内部自动执行的完整流程

`write_article.js` 是唯一入口，内部自动编排以下步骤，**不允许跳过任何一步**：

1. **公司概况底稿** → `generate_company_profile_brief.py`
2. **内部风险底稿** → `generate_risk_brief.py`（基于 financial_data.json）
3. **外部风险底稿** → `generate_external_risk_brief.py`（基于 Tavily 外部检索）
4. **合并底稿** → 内部 + 外部风险底稿合并
5. **初稿生成** → `bailian_writer.py`（基于底稿 + SKILL 规范写稿）
6. **强制审稿** → `review_article.py`（机械比对财务数据，不通过则返修，最多 3 轮）
7. **去模板化** → `bailian_writer.py humanize`（去除 AI 味）
8. **AI 配图** → `generate_images.js`（4 张，ModelScope）
9. **推草稿箱** → `publish_draft.js` → 调用 `publish_to_wechat.js` 推送到微信公众号草稿箱
10. **ZIP 打包** → `package_output.js`（包含 article.md、article.html、**financial_analysis.md**、company_profile_brief.md、risk_brief.md、external_risk_brief.md、generation_status.json、financial_data.json、images/）

### 明确禁止

- ❌ **禁止**直接调用 `bailian_writer.py` 跳过低稿生成
- ❌ **禁止**跳过 `generate_company_profile_brief.py`、`generate_risk_brief.py`、`generate_external_risk_brief.py`
- ❌ **禁止**手动分步执行（先写稿、再配图、再推草稿箱）
- ❌ **禁止**省略审稿环节
- ❌ **禁止**省略 ZIP 打包环节
- ❌ **禁止手写 zip/cp 命令自行打包**（必须调用 `node scripts/package_output.js` 或通过 `write_article.js` 内部自动调用，防止漏打 financial_analysis.md 等分析文件）
- ✅ **唯一合法入口**：`node scripts/write_article.js <公司名称> <股票代码> <输出目录>`（至少对 A 股/港股上市公司，必须显式传股票代码）

## 十、朋友圈宣传文案

每期文章推送到草稿箱后，**必须生成一段朋友圈宣传文案**发给用户。

### 风格：悬念型（固定）

以最反直觉或最震撼的数据/事实开头，制造"不点开不行"的冲动。

### 模板结构

```
[第 1 段] 反直觉的开头 / 最震撼的数据对比
     → 用 1-2 句话制造悬念或冲击感

[第 2 段] 3-5 条触目惊心的财务数据
     → 每条用 emoji 开头（📉⚠️🔴💰）
     → 只挑最扎心的数据，不要面面俱到

[第 3 段] 杜邦分析或风控模块的"金句"
     → 引用文章中最犀利的一句判断

[第 4 段] 争议性的提问 + 引导阅读
     → 用一个问题结尾，引发好奇
     → 最后一行：公司名｜一句话定位👇
```

### 写作要求

- **总字数**：150-250 字（朋友圈最佳阅读长度）
- **数据密度**：至少包含 3 个具体数字
- **不要写成广告**：要像在朋友圈分享一个"发现"，而不是推销文章
- **必须包含文章的核心论点**（一句话总结）
- **结尾引导**：用问句 + 👇 emoji 引导点击

### 示例（格林美）

> 负债率从 52% 飙到 65%，流动比率跌破 1，大股东减持量是增持量的 15 倍……
>
> 格林美，一家踩在"电池回收"黄金赛道上的公司，财报里却藏着这么多让人冒冷汗的数字。
>

> ✅ 三年趋势对比（不看一个点，看方向）
> ✅ 杜邦分解（ROE 靠利润还是靠借钱？）
> ✅ 利润质量检验（赚的是真金白银还是纸面富贵？）
